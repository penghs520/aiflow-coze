# AI工作流平台后端项目结构说明

## 项目概述
- **项目名称**: AI工作流平台后端
- **技术栈**: SpringBoot 3.2.0 + Java 17 + MySQL 8.0 + Redis 7.0
- **构建工具**: Maven
- **代码结构**: 分层架构（Entity, Repository, Service, Controller）

## 实体类设计说明

### 1. User（用户）
- **对应表**: `users`
- **核心字段**: `id`(自增主键), `phone`(唯一), `nickname`, `points_balance`(默认30000)
- **状态定义**: 0-禁用, 1-正常, 2-锁定
- **业务方法**: 余额检查、点数增减、状态判断

### 2. Workflow（工作流）
- **对应表**: `workflows`
- **核心字段**: `id`(字符串主键), `name`, `category`, `parameter_definition`(JSON)
- **状态定义**: 0-下架, 1-上架
- **JSON字段**: `tags`(标签数组), `parameter_definition`(参数定义)
- **业务方法**: 状态判断、使用次数递增、评分更新

### 3. Task（任务）
- **对应表**: `tasks`
- **核心字段**: `id`(字符串主键), `workflow_id`, `user_id`, `status`, `parameters`(JSON)
- **状态定义**: 0-待提交, 1-排队中, 2-处理中, 3-已完成, 4-失败, 5-已取消
- **JSON字段**: `parameters`(任务参数), `result`(执行结果)
- **业务方法**: 状态流转、进度更新、取消/重试判断

### 4. RechargeOrder（充值订单）
- **对应表**: `orders`
- **核心字段**: `id`(字符串主键), `user_id`, `amount`, `points`, `status`
- **订单类型**: 1-资源点充值
- **状态定义**: 0-待支付, 1-支付成功, 2-支付失败, 3-已退款
- **业务方法**: 支付状态更新、退款处理

### 5. PointRecord（资源点记录）
- **对应表**: `point_records`
- **核心字段**: `id`(自增), `user_id`, `change_type`, `change_amount`, `balance_after`
- **变动类型**: 1-注册赠送, 2-充值获得, 3-任务消耗, 4-退款返还
- **业务方法**: 变动类型文本转换、记录创建工厂方法

## Repository接口说明

### 1. UserRepository
- 支持按手机号、状态查询
- 支持昵称模糊搜索
- 包含统计和更新方法

### 2. WorkflowRepository
- 支持按分类、状态查询
- 支持标签搜索（JSON_CONTAINS）
- 支持关键词搜索和排序

### 3. TaskRepository
- 支持按用户、工作流、状态查询
- 支持分页和范围查询
- 包含超时任务和状态同步查询

### 4. RechargeOrderRepository
- 支持按用户、状态、支付渠道查询
- 包含金额统计和超时订单查询

### 5. PointRecordRepository
- 支持按用户、变动类型查询
- 包含各种统计汇总方法

## 技术特点

### 1. JPA注解规范
- 使用`@Entity`、`@Table`定义实体映射
- `@Id`、`@GeneratedValue`定义主键策略
- `@Column`定义字段属性
- `@CreationTimestamp`、`@UpdateTimestamp`自动维护时间戳
- `@JdbcTypeCode(SqlTypes.JSON)`支持JSON字段

### 2. Lombok集成
- 使用`@Data`、`@Builder`、`@NoArgsConstructor`、`@AllArgsConstructor`
- 减少样板代码，提高开发效率

### 3. 索引优化
- 根据业务查询模式设计合理索引
- 组合索引提升多条件查询性能

### 4. 业务方法封装
- 实体类包含领域逻辑，符合DDD思想
- 状态流转封装，保证业务一致性

## 部署运行

### 环境要求
1. Java 17+
2. Maven 3.6+
3. MySQL 8.0+
4. Redis 7.0+

### 启动步骤
```bash
# 1. 进入项目目录
cd src/backend

# 2. 编译项目
mvn clean compile

# 3. 运行应用
mvn spring-boot:run

# 或直接运行jar包
mvn package
java -jar target/backend-1.0.0.jar
```

### 数据库初始化
- 首次启动时JPA会自动创建表结构（ddl-auto: update）
- 确保MySQL服务运行并创建`ai_workflow`数据库
