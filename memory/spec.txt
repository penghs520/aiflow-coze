# AI工作流平台App开发 工作计划
## 1. 核心目标
### 目标概述
开发一款可上线的完整 Android App，作为扣子平台工作流的展示与使用客户端。用户可在 App 中浏览、预览并运行已配置的视频创作类、图文创作类工作流，提交任务后后台执行，任务完成后通知用户查看结果。平台需支持用户注册、充值资源点（首次注册赠送30000点），并提供一个管理后台用于配置上线工作流。

### 背景与动机
参考米核平台（https://miheai.com/#/ai）的商用AI模块，在扣子中发布视频/图文创作工作流，通过 App 为用户提供便捷的使用界面。团队拥有后端开发能力（Java/SpringBoot），缺少前端开发能力，因此采用 Android 原生开发，UI 风格参考即梦 App 的深色配色与原生质感。

## 2. 要求与标准
- 产出要求：
  - 产品需求文档（PRD）：完整的功能清单、交互逻辑、信息架构，明确工作流展示、参数输入、任务提交、后台配置等核心流程
  - 高保真可交互原型：基于即梦 App 的深色风格，绘制 App 各页面的视觉与交互原型，用于开发参考

  - 技术方案文档：前后端技术选型、API 接口设计、数据库 schema、部署方案、与扣子工作流的集成方案
  - 后台管理系统：提供工作流配置后台，支持动态上线/下线工作流，配置工作流参数、资源点消耗等
  - Android App：实现工作流展示、预览、参数输入、任务提交、任务中心、用户注册登录、资源点充值等核心功能
  - 后端代码：基于 SpringBoot 实现完整的后端 API，包括用户认证、工作流管理、任务管理、资源点管理、第三方服务集成等
  - 后台管理前端：基于 Vue 3 + Element Plus 实现工作流配置、用户管理、任务监控等后台管理界面
- 风格偏好：
  - UI 风格：参考即梦 App 的深色原生风格，配色、字体、组件样式保持一致的现代感与专业感
  - 交互逻辑：符合 Android 设计规范，操作流畅，信息层级清晰
- 文件目录规则：
  - 过程数据与缓存：`temp/`
  - 核心交付资产：`outputs/`（包括 PRD、原型、技术文档、测试报告等）
  - 逻辑脚本与工具：`scripts/`（如有自动化脚本）
  - 数据文件：`data/`（如测试数据、配置数据）
- 数据依赖：
  - 用户上传的米核平台界面截图（`image.png`、`image(1).png`）
  - 用户上传的即梦 App 界面截图（`644ff30c3acbf3466b5ad3c52cf4474c.jpg`）
  - 技术方案文档：`outputs/技术方案/AI工作流平台技术方案文档.md`（包含API设计、数据库schema、部署方案等）
- 通知规则：
  - 关键里程碑完成后（如 PRD 完成、原型完成、工作流验证通过）需通知用户
  - 任务执行中出现阻塞或需用户决策的问题时需及时通知
