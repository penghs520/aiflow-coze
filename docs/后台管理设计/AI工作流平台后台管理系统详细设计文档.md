## 1. 项目概述

### 1.1 项目背景
AI工作流平台是一个提供视频/图文创作工作流的服务平台，用户可通过移动端App（iOS/Android）浏览、预览并运行已配置的工作流。后台管理系统作为平台的管理中枢，负责工作流配置、用户管理、任务监控、数据统计等核心管理功能。

### 1.2 设计目标
1. **可管理性**：支持上百个工作流的动态配置与管理
2. **安全性**：基于角色的精细化权限控制，操作全程审计
3. **可扩展性**：模块化设计，便于功能扩展和性能优化
4. **用户体验**：直观易用的管理界面，高效的操作流程

### 1.3 技术栈
| 组件 | 技术方案 | 版本 | 说明 |
|------|---------|------|------|
| **前端** | Vue 3 + Element Plus | Vue 3.3+, Element Plus 2.4+ | 渐进式框架，组件丰富 |
| **后端** | SpringBoot 3.x + Java 17 | SpringBoot 3.2.0+, Java 17 LTS | 团队熟悉，生产级稳定性 |
| **数据库** | MySQL 8.0 + Redis 7.0 | MySQL 8.0+, Redis 7.0+ | 关系型存储+缓存加速 |
| **部署** | Docker + Kubernetes | Docker 24+, K8s 1.28+ | 容器化，高可用部署 |

## 2. 系统架构设计

### 2.1 整体架构
后台管理系统采用前后端分离架构，独立部署在子域名下，与主业务系统共享核心数据服务但逻辑隔离。

```
架构层次：
┌─────────────────────────────────┐
│        客户端层 (Client)         │
│   ┌─────────────────────┐       │
│   │  Vue 3 + Element Plus│       │
│   │  后台管理前端应用    │       │
│   └─────────────────────┘       │
├─────────────────────────────────┤
│        接入层 (Gateway)          │
│   ┌─────────────────────┐       │
│   │  Spring Cloud Gateway│       │
│   │  API网关 + 负载均衡  │       │
│   └─────────────────────┘       │
├─────────────────────────────────┤
│        应用层 (Application)      │
│   ┌─────────────────────┐       │
│   │  SpringBoot 3.x     │       │
│   │  后台管理业务逻辑    │       │
│   └─────────────────────┘       │
├─────────────────────────────────┤
│        数据层 (Data)             │
│   ┌─────┐  ┌─────┐  ┌─────┐    │
│   │MySQL│  │Redis│  │ OSS │    │
│   └─────┘  └─────┘  └─────┘    │
└─────────────────────────────────┘
```

详细架构图见：[后台管理系统架构图](后台管理系统架构图.drawio)

### 2.2 模块划分
| 模块名称 | 职责描述 | 主要功能 |
|---------|---------|---------|
| **认证授权模块** | 管理员身份认证与权限验证 | 登录、登出、Token刷新、权限拦截 |
| **工作流管理模块** | 工作流生命周期管理 | 创建、编辑、上下架、参数配置、统计 |
| **用户管理模块** | 平台用户账户管理 | 用户列表、状态管理、资源点调整 |
| **任务监控模块** | 任务执行过程监控 | 任务状态跟踪、失败重试、执行日志 |
| **系统管理模块** | 平台基础配置管理 | 角色管理、操作日志、系统配置、通知公告 |

### 2.3 技术架构特性
1. **微服务就绪**：模块化设计，便于未来拆分为微服务
2. **缓存策略**：多级缓存设计，提升系统响应速度
3. **异步处理**：耗时操作异步化，提升用户体验
4. **监控告警**：全方位监控体系，保障系统稳定运行

## 3. 数据库设计

### 3.1 数据库选型
- **主数据库**：MySQL 8.0（InnoDB引擎，ROW格式）
- **缓存数据库**：Redis 7.0（哨兵模式或集群）
- **文件存储**：阿里云OSS（对象存储）

### 3.2 核心业务表扩展
在原有业务表基础上，后台管理系统新增以下管理表：

1. **管理员表 (admin_users)**：后台管理员账户信息
2. **角色表 (admin_roles)**：管理员角色定义
3. **权限表 (admin_permissions)**：细粒度操作权限
4. **角色权限关联表 (admin_role_permissions)**：角色与权限多对多关系
5. **操作日志表 (admin_operation_logs)**：管理员操作审计记录
6. **系统配置表 (system_configs)**：平台运行参数配置
7. **通知公告表 (system_notices)**：系统公告管理

### 3.3 数据权限设计
采用RBAC（基于角色的访问控制）模型，实现：
1. **角色权限分离**：不同角色拥有不同操作权限
2. **数据访问控制**：基于角色限制数据访问范围
3. **操作审计追踪**：完整记录所有管理操作

详细数据库设计见：[数据库设计细化](数据库设计细化.md)

## 4. API接口设计

### 4.1 API规范
- **协议**：HTTPS 1.1/2.0
- **格式**：JSON（Content-Type: application/json）
- **认证**：JWT Token（Bearer方案）
- **前缀**：`/api/admin/v1/`
- **响应**：统一包装结构，包含code、message、data字段

### 4.2 核心接口模块
1. **认证授权接口**：登录、登出、刷新Token、修改密码
2. **管理员管理接口**：管理员CRUD、角色分配、状态管理
3. **工作流管理接口**：工作流配置、上下架、参数管理、统计分析
4. **用户管理接口**：用户列表、详情查看、状态调整、资源点管理
5. **任务监控接口**：任务列表、状态跟踪、失败重试、日志查询
6. **系统管理接口**：角色权限管理、操作日志查询、系统配置、通知公告

### 4.3 接口安全设计
1. **权限验证**：基于角色的细粒度权限控制
2. **请求限流**：防止恶意请求和DDoS攻击
3. **数据脱敏**：敏感信息返回时进行脱敏处理
4. **操作审计**：所有操作记录完整日志

详细API接口设计见：[API接口设计文档](API接口设计文档.md)

## 5. 前端界面设计

### 5.1 设计原则
1. **一致性**：所有页面保持统一的设计语言和交互模式
2. **可用性**：操作流程清晰，降低用户学习成本
3. **响应式**：支持桌面端、平板端和移动端适配
4. **性能优先**：优化加载速度，提升用户体验

### 5.2 核心页面设计
1. **登录页**：简洁专业的登录表单，支持验证码防刷
2. **主布局框架**：经典后台布局（侧边菜单+顶部导航+内容区）
3. **仪表盘**：数据概览卡片、趋势图表、实时监控面板
4. **工作流管理**：列表+详情+表单三合一视图，支持批量操作
5. **用户管理**：用户列表与详情，状态管理和资源点调整
6. **系统管理**：角色权限配置、操作日志查询、系统参数设置

### 5.3 组件库设计
1. **数据表格组件**：支持分页、排序、筛选、批量操作
2. **表单组件**：统一验证规则，支持动态表单
3. **搜索组件**：多条件组合搜索，支持快捷筛选
4. **上传组件**：支持图片预览、裁剪、格式验证

详细前端界面设计见：[前端界面设计文档](前端界面设计文档.md)

## 6. 权限控制方案

### 6.1 RBAC模型设计
采用标准RBAC模型，实现：
- **管理员** ↔ **角色**：多对一关系
- **角色** ↔ **权限**：多对多关系
- **权限** ↔ **操作**：细粒度控制

### 6.2 系统内置角色
1. **超级管理员 (super_admin)**：所有权限，系统最高管理者
2. **工作流管理员 (workflow_manager)**：工作流相关所有权限
3. **用户管理员 (user_manager)**：用户管理相关权限
4. **财务管理员 (finance_manager)**：订单和资源点管理权限
5. **数据观察员 (data_viewer)**：只读权限，可查看数据统计

### 6.3 权限验证流程
1. **前端路由守卫**：检查页面访问权限
2. **后端接口拦截**：验证接口操作权限
3. **按钮级控制**：控制功能按钮显示和操作
4. **数据权限过滤**：基于角色过滤数据访问范围

详细权限控制方案见：[权限控制方案](权限控制方案.md)

## 7. 部署与运维方案

### 7.1 部署架构
采用容器化部署方案，支持多环境部署：
- **开发环境**：本地Docker Compose部署
- **测试环境**：独立服务器部署
- **预发环境**：准生产环境配置
- **生产环境**：高可用集群部署

### 7.2 CI/CD流程
实现自动化构建、测试和部署：
- **代码提交** → **代码检查** → **单元测试** → **构建镜像**
- **安全扫描** → **测试部署** → **人工审批** → **生产部署**

### 7.3 监控告警体系
建立全方位监控体系：
1. **基础设施监控**：服务器、网络、存储状态
2. **应用性能监控**：JVM、API响应、业务指标
3. **日志分析系统**：集中日志收集、实时分析
4. **告警通知机制**：分级告警，多渠道通知

### 7.4 备份恢复策略
确保数据安全与业务连续性：
1. **定期备份**：数据库每日全备+实时增量
2. **备份验证**：定期验证备份数据完整性
3. **恢复演练**：每季度进行数据恢复演练
4. **应急预案**：制定详细故障处理流程

详细部署与运维方案见：[部署与运维方案](部署与运维方案.md)

## 8. 安全防护措施

### 8.1 网络安全
1. **VPC隔离**：生产环境独立网络隔离
2. **WAF防护**：Web应用防火墙防护
3. **DDoS防护**：流量清洗与防护
4. **访问控制**：IP白名单，VPN访问

### 8.2 应用安全
1. **输入验证**：严格验证所有用户输入
2. **输出编码**：防止XSS跨站脚本攻击
3. **SQL防注入**：使用预编译语句
4. **CSRF防护**：令牌验证防止跨站请求伪造

### 8.3 数据安全
1. **加密存储**：敏感数据加密存储
2. **访问审计**：所有数据访问操作记录
3. **数据脱敏**：日志和导出数据脱敏处理
4. **权限分离**：基于角色的数据访问控制

## 9. 性能优化策略

### 9.1 前端优化
1. **代码分割**：路由懒加载，组件异步加载
2. **资源压缩**：图片、CSS、JS文件压缩
3. **缓存策略**：静态资源长期缓存
4. **按需加载**：第三方库按需引入

### 9.2 后端优化
1. **数据库优化**：索引优化，查询重写，读写分离
2. **缓存应用**：多级缓存，热点数据预加载
3. **异步处理**：耗时操作异步化，队列缓冲
4. **连接池优化**：合理配置数据库连接池

### 9.3 部署优化
1. **CDN加速**：静态资源全球加速
2. **负载均衡**：多实例负载，自动扩缩容
3. **容器优化**：资源限制，健康检查
4. **监控调优**：实时监控，自动告警
