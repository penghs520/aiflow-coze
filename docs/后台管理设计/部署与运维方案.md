# 后台管理系统部署与运维方案

## 1. 部署架构概述

### 1.1 整体架构
后台管理系统采用前后端分离架构，部署在独立的子域名下，与主业务系统隔离但共享核心数据服务。

```
部署架构图：
┌─────────────────────────────────────────────────────────────┐
│                     admin.workflow-app.com                   │
│                    (后台管理独立域名)                         │
├─────────────────────────────────────────────────────────────┤
│                       负载均衡器 (Nginx)                      │
│                    SSL终止 + 静态文件服务                     │
├──────────────────────────┬──────────────────────────────────┤
│    前端资源 (Vue)        │       后端API (SpringBoot)       │
│   admin-frontend-vue     │      admin-backend-service       │
│   Docker容器/VPS部署      │      Docker容器/VPS部署          │
└──────────────────────────┴──────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   共享数据服务层                               │
│         MySQL集群 + Redis集群 + 对象存储(OSS)                 │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 部署环境规划
| 环境 | 访问域名 | 用途 | 数据隔离 | 部署方式 |
|------|---------|------|---------|---------|
| 开发环境 | admin-dev.workflow-app.com | 开发自测、功能验证 | 开发数据库 | Docker Compose本地部署 |
| 测试环境 | admin-test.workflow-app.com | 集成测试、UAT验收 | 测试数据库 | 测试服务器部署 |
| 预发环境 | admin-staging.workflow-app.com | 上线前验证、压力测试 | 独立准生产数据库 | 生产环境相同配置 |
| 生产环境 | admin.workflow-app.com | 线上正式服务 | 生产数据库 | 高可用集群部署 |

## 2. 基础设施要求

### 2.1 服务器配置
| 组件 | 生产环境配置 | 数量 | 说明 |
|------|-------------|------|------|
| **前端服务器** | 2核4G内存，50GB SSD | 2台 | 负载均衡，高可用 |
| **后端应用服务器** | 4核8G内存，100GB SSD | 2-4台 | 根据业务量弹性扩展 |
| **数据库服务器** | 8核16G内存，500GB SSD RAID10 | 1主2从 | MySQL主从复制，读写分离 |
| **缓存服务器** | 4核8G内存，100GB SSD | 2-3节点 | Redis哨兵模式或集群 |
| **负载均衡器** | 4核4G内存 | 2台 | Nginx + Keepalived高可用 |

### 2.2 网络与安全
1. **网络拓扑**：
   - VPC网络隔离，安全组最小权限原则
   - 公网访问仅开放80/443端口
   - 内网通信使用私有IP，端口按需开放

2. **域名与证书**：
   - 独立子域名：admin.workflow-app.com
   - SSL证书：Let's Encrypt免费证书或商业证书
   - 自动续期：使用Certbot工具自动化管理

3. **防火墙规则**：
   - 仅允许特定IP段访问后台管理系统（办公网络IP）
   - 禁止境外IP访问（根据业务需求调整）
   - 配置DDoS防护（云服务商提供或自建）

## 3. 容器化部署方案

### 3.1 Docker镜像配置

#### 3.1.1 前端镜像 (Dockerfile)
```dockerfile
# 前端Vue应用Dockerfile
FROM node:18-alpine as builder

WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

# 生产阶段
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:80/ || exit 1

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

#### 3.1.2 后端镜像 (Dockerfile)
```dockerfile
# SpringBoot应用Dockerfile
FROM openjdk:17-jdk-slim as builder

WORKDIR /app
COPY mvnw .
COPY .mvn .mvn
COPY pom.xml .
RUN ./mvnw dependency:go-offline

COPY src src
RUN ./mvnw clean package -DskipTests

# 生产阶段
FROM openjdk:17-jdk-slim

WORKDIR /app
COPY --from=builder /app/target/*.jar app.jar

# 时区设置
RUN ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

# JVM参数
ENV JAVA_OPTS="-Xms512m -Xmx1024m -XX:+UseG1GC -XX:MaxGCPauseMillis=200"

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=60s --retries=3 \
  CMD curl -f http://localhost:8080/actuator/health || exit 1

EXPOSE 8080
ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar app.jar"]
```

### 3.3 Nginx配置

#### 3.3.1 生产环境Nginx配置 (nginx.conf)
```nginx
# 全局配置
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;
    multi_accept on;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    # 日志格式
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    # 基础优化
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;
    client_max_body_size 100M;

    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_comp_level 6;
    gzip_types text/plain text/css text/xml text/javascript 
               application/json application/javascript application/xml+rss 
               application/atom+xml image/svg+xml;

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # 后台管理系统配置
    server {
        listen 80;
        server_name admin.workflow-app.com;
        return 301 https://$server_name$request_uri;
    }

    server {
        listen 443 ssl http2;
        server_name admin.workflow-app.com;

        # SSL证书
        ssl_certificate /etc/nginx/ssl/fullchain.pem;
        ssl_certificate_key /etc/nginx/ssl/privkey.pem;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;
        ssl_prefer_server_ciphers off;
        ssl_session_cache shared:SSL:10m;
        ssl_session_timeout 10m;

        # 前端静态资源
        location / {
            root /usr/share/nginx/html;
            index index.html;
            try_files $uri $uri/ /index.html;
            
            # 缓存策略
            location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
                expires 1y;
                add_header Cache-Control "public, immutable";
            }
        }

        # 后端API代理
        location /api/ {
            proxy_pass http://backend:8080;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # 超时设置
            proxy_connect_timeout 30s;
            proxy_send_timeout 60s;
            proxy_read_timeout 60s;
            
            # 开启WebSocket支持
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
        }

        # 健康检查端点
        location /health {
            access_log off;
            return 200 "healthy\n";
            add_header Content-Type text/plain;
        }
    }
}
```

## 4. CI/CD流水线设计

### 4.1 整体流程
```
代码提交 → 代码扫描 → 单元测试 → 构建镜像 → 安全扫描 → 部署测试 → 人工审批 → 部署生产
```

### 4.2 Jenkins Pipeline脚本 (Jenkinsfile)
```groovy
pipeline {
    agent any
    
    environment {
        DOCKER_REGISTRY = 'registry.example.com'
        PROJECT_NAME = 'ai-workflow-admin'
        KUBE_CONFIG = credentials('kube-config')
    }
    
    stages {
        // 阶段1: 代码检查
        stage('Code Check') {
            steps {
                sh '''
                    echo "进行代码检查..."
                    npm run lint  # 前端
                    ./mvnw checkstyle:check  # 后端
                '''
            }
        }
        
        // 阶段2: 单元测试
        stage('Unit Test') {
            steps {
                sh '''
                    echo "运行单元测试..."
                    npm run test:unit  # 前端
                    ./mvnw test  # 后端
                '''
            }
            post {
                always {
                    junit '**/target/surefire-reports/*.xml'
                }
            }
        }
        
        // 阶段3: 构建镜像
        stage('Build Docker Image') {
            parallel {
                stage('Build Frontend') {
                    steps {
                        sh '''
                            cd frontend
                            docker build -t ${DOCKER_REGISTRY}/${PROJECT_NAME}-frontend:${BUILD_NUMBER} .
                        '''
                    }
                }
                stage('Build Backend') {
                    steps {
                        sh '''
                            cd backend
                            docker build -t ${DOCKER_REGISTRY}/${PROJECT_NAME}-backend:${BUILD_NUMBER} .
                        '''
                    }
                }
            }
        }
        
        // 阶段4: 安全扫描
        stage('Security Scan') {
            steps {
                sh '''
                    echo "进行安全漏洞扫描..."
                    # 使用Trivy扫描镜像
                    trivy image ${DOCKER_REGISTRY}/${PROJECT_NAME}-backend:${BUILD_NUMBER}
                    trivy image ${DOCKER_REGISTRY}/${PROJECT_NAME}-frontend:${BUILD_NUMBER}
                '''
            }
        }
        
        // 阶段5: 推送到镜像仓库
        stage('Push to Registry') {
            steps {
                sh '''
                    docker push ${DOCKER_REGISTRY}/${PROJECT_NAME}-frontend:${BUILD_NUMBER}
                    docker push ${DOCKER_REGISTRY}/${PROJECT_NAME}-backend:${BUILD_NUMBER}
                '''
            }
        }
        
        // 阶段6: 部署到测试环境
        stage('Deploy to Test') {
            when {
                branch 'develop'
            }
            steps {
                sh '''
                    echo "部署到测试环境..."
                    kubectl --kubeconfig=${KUBE_CONFIG} \
                        set image deployment/admin-frontend \
                        frontend=${DOCKER_REGISTRY}/${PROJECT_NAME}-frontend:${BUILD_NUMBER}
                    kubectl --kubeconfig=${KUBE_CONFIG} \
                        set image deployment/admin-backend \
                        backend=${DOCKER_REGISTRY}/${PROJECT_NAME}-backend:${BUILD_NUMBER}
                '''
            }
        }
        
        // 阶段7: 部署到生产环境
        stage('Deploy to Production') {
            when {
                branch 'main'
            }
            steps {
                input message: '确认部署到生产环境?', ok: '确认部署'
                sh '''
                    echo "部署到生产环境..."
                    # 蓝绿部署策略
                    kubectl --kubeconfig=${KUBE_CONFIG} apply -f k8s/production/
                '''
            }
        }
    }
    
    post {
        success {
            echo '构建部署成功!'
        }
        failure {
            echo '构建部署失败!'
        }
        always {
            echo '清理工作空间...'
            cleanWs()
        }
    }
}
```

### 4.3 Git分支策略
```
main分支       -- 生产环境代码，保护分支，只接受merge request
develop分支    -- 集成测试环境，功能开发完成后的合并分支
feature/*分支  -- 功能开发分支，从develop创建，开发完成后合并到develop
hotfix/*分支   -- 紧急修复分支，从main创建，修复后合并到main和develop
```

|---------|---------|---------|---------|
| JVM | 堆内存、GC时间、线程数 | 堆使用>85%, Full GC频繁 | Spring Boot Actuator, Micrometer |
| API | 响应时间、错误率、QPS | P95>1s, 错误率>1% | Prometheus, Grafana |

### 5.2 告警配置

#### 5.2.1 告警规则 (Prometheus)
```yaml
groups:
  - name: backend-alerts
    rules:
      - alert: HighErrorRate
        expr: rate(http_server_requests_seconds_count{status!~"2.."}[5m]) / rate(http_server_requests_seconds_count[5m]) > 0.01
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "高错误率告警"
          description: "错误率超过1%，当前值: {{ $value }}"
          
      - alert: HighResponseTime
        expr: histogram_quantile(0.95, rate(http_server_requests_seconds_bucket[5m])) > 1
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "高响应时间告警"
          description: "P95响应时间超过1s，当前值: {{ $value }}s"
```

## 6. 备份与恢复策略

### 6.1 数据备份方案

#### 6.1.1 MySQL备份
```bash
#!/bin/bash
# mysql-backup.sh

# 全量备份
mysqldump -h $DB_HOST -u $DB_USER -p$DB_PASSWORD \
  --single-transaction \
  --routines \
  --triggers \
  --events \
  --all-databases \
  > /backup/mysql/full-$(date +%Y%m%d).sql

# 备份压缩
gzip /backup/mysql/full-$(date +%Y%m%d).sql

# 保留最近30天的备份
find /backup/mysql -name "*.sql.gz" -mtime +30 -delete
```

#### 6.1.2 Redis备份
```bash
#!/bin/bash
# redis-backup.sh

# RDB备份
redis-cli -h $REDIS_HOST save
cp /var/lib/redis/dump.rdb /backup/redis/dump-$(date +%Y%m%d).rdb

# AOF备份
cp /var/lib/redis/appendonly.aof /backup/redis/appendonly-$(date +%Y%m%d).aof
```

## 7. 安全防护措施

### 7.1 网络安全
1. **VPC隔离**：生产环境独立VPC，安全组最小权限原则
2. **WAF防护**：部署Web应用防火墙，防护SQL注入、XSS等攻击
3. **DDoS防护**：启用云服务商DDoS防护服务，阈值告警
4. **VPN访问**：后台管理仅允许通过VPN访问（办公网络IP白名单）

### 7.2 应用安全
1. **输入验证**：所有用户输入进行严格验证和过滤
2. **输出编码**：防止XSS攻击，对输出内容进行编码
3. **SQL防注入**：使用预编译语句，MyBatis参数绑定
4. **CSRF防护**：Spring Security默认启用CSRF防护
5. **Session安全**：设置HttpOnly、Secure属性

### 7.3 数据安全
1. **敏感数据加密**：密码使用BCrypt哈希，支付信息加密存储
2. **数据脱敏**：日志中敏感信息脱敏处理
3. **访问审计**：所有数据访问操作记录审计日志

## 8. 应急预案

### 8.1 常见故障处理

#### 8.1.1 数据库故障
```
故障现象：数据库连接失败，应用无法启动
处理步骤：
1. 检查数据库服务状态
2. 查看数据库日志，定位问题原因
3. 如主库故障，切换到从库
4. 启动数据库备份恢复流程
```

#### 8.1.2 应用故障
```
故障现象：API响应超时或返回错误
处理步骤：
1. 检查应用日志，定位异常堆栈
2. 查看JVM监控，判断是否内存溢出
3. 重启应用实例
4. 如问题持续，回滚到上一个稳定版本
```

### 9.2 版本更新流程
```
1. 开发完成 → 代码评审 → 合并到develop分支
2. Jenkins自动构建 → 部署到测试环境
3. 测试验证 → 生成测试报告
4. 发布审批 → 合并到main分支
5. 生产环境部署 → 蓝绿发布/滚动更新
6. 监控观察 → 确认无误，更新完成
```

### 9.3 数据迁移方案
1. **停机迁移**：小数据量，停机时间可接受时使用
2. **双写迁移**：大数据量，要求零停机时间
3. **增量迁移**：持续同步，逐步切换
